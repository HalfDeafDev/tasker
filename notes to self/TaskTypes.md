# Task Types

## Habit
A recurring behavior where completion 
history/streak/rhythm matters more than 
producing independent scheduled tasks.

A behavior the user is trying to maintain
or measure over time, where completion rhythm
matters more than the individual task instance.

- Streak Breakpoint (DateTime)
- Streak Count
- Top Streak
  - DateTime
  - Streak Count

## Recurring
A task that produces instances based
on calendar/time/completion rules. Example:
"clean filter 30 days after I last did it",
"every Monday at 8 AM".

A task whose next instance can be generated from a known schedule.

## Repeating
A task that produces the next occurrence
based on completion or some event. Example:
"I want to keep track of how often I need to cook"

A task expected to happen again, but whose next instance
depends on condition, demand, completion history, or user judgment.
